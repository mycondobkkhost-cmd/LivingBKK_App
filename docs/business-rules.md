# LivingBKK — Business Rules

เอกสารอ้างอิงกฎธุรกิจหลัก (อัปเดตตามการออกแบบ Phase 1–2)

## 1. ขอบเขตและโมเดล

- พื้นที่: กรุงเทพฯ + ปริมณฑล เท่านั้น
- ไม่มี booking รายวัน
- ผู้ใช้ทั่วไปและเอเจ้นท์ **ไม่เห็นเบอร์โทรหรือ Line ID** ของผู้ลงประกาศ
- **โพสต์ประกาศฟรี** สำหรับ Owner / Agent (ไม่มีแพ็ก subscription)
- รายได้แพลตฟอร์ม: Success Fee (คอมขั้นบันไดตามสัญญา) เท่านั้น

## 2. การลงประกาศ

- Owner และ Agent **ลงประกาศได้ฟรี** — ไม่มีค่าสมาชิกรายปีหรือแพ็ก PM subscription
- **บัญชีเดียว** สลับมุมมอง ลูกค้า / เอเจ้นท์ / เจ้าของ ที่หน้าแรก (แอดมินระบบแยก)

## 3. ราคา

- ผู้ลงประกาศกรอกราคา **รวมค่าคอมมิชชัน 100% แล้ว**
- ผู้เข้าชมเห็นเฉพาะ **ราคา Net (Final Price)** — ห้ามแสดงส่วนแบ่งคอมฝั่งผู้เช่า/ซื้อ

## 4. ความเป็นส่วนตัวทรัพย์

- บน Property Card และรายละเอียด: **ซ่อนเลขห้องและชั้นที่แน่นอน**
- แผนที่สาธารณะ: ใช้ pin/โซนแบบ approximate (offset)

## 5. Co-Agent (ทรัพย์ในระบบ)

ทรัพย์ที่เอเจ้นท์ **ขอโคเอเจ้นท์กับ LivingBKK ได้** เมื่อ:

- เจ้าของโพสต์เอง (Owner Direct, verified) **หรือ**
- แพลตฟอร์มมีคอนแทคเจ้าของโดยตรง + เจ้าของ opt-in รับ co-agent

เอเจ้นท์มีมุมมองแผนที่/ฟิลเตอร์ **「ขอโคเอเจ้นท์ได้」** แยกจากผู้ใช้ทั่วไป

## 6. Demand Board (บอร์ดประกาศ)

- Admin/LivingBKK สร้าง `demand_post` (เช่น หาคอนโดทองหล่อ ≤15k)
- **Seeker, Owner, Agent** เห็นรายการบอร์ดได้
- ผู้เสนอทรัพย์: ปุ่ม **เสนอทรัพย์** — ลงในแอป (รูป+รายละเอียด) หรือแปะลิงก์ภายนอก (เช่น Facebook)
- **บังคับ dropdown** ประกาศในฐานะ:
  - `owner_direct_100` — เจ้าของทรัพย์ (Owner 100%)
  - `co_agent_50_50` — โคเอเจ้นท์ 50/50
  - `listing_agent` — เอเจ้นท์ฝั่งประกาศ (รอตรวจ)
- **ผู้ใช้รายอื่นไม่เห็น** ว่าใครเสนออะไร — เฉพาะศูนย์กลาง (Admin) และผู้เสนอเห็นสถานะของตัวเอง

## 7. Lead & สัญญา

- Lead Bot บังคับเก็บฟิลด์คัดกรองก่อนนัด
- Push ไป Owner/Agent: โปรไฟล์ **เบอร์ censored**
- รับเคส: บังคับ **E-Contract** ยอมรับโครงสร้างคอมก่อนเริ่มงาน
- ทรัพย์ไม่ว่าง: ระบุวันสัญญาถึงเมื่อไหร่ / ว่างอีกเมื่อไหร่

## 8. Moderation & Lifecycle

- สแกนคำอธิบาย: เบอร์, Line, ลิงก์นอกระบบ → แจ้งเตือน/ระงับ publish
- รูปซ้ำ: AI dedup → Admin อนุมัติ
- ประกาศมีอายุ: ยืนยันว่าง = bump; ไม่ยืนยัน = ซ่อนอัตโนมัติ

## 9. ทะเบียนทรัพย์รวม (RXT) — ประกาศซ้ำ

- หลายเอเจ้นท์ลงทรัพย์เดียวกันได้ — ลูกค้าเห็น **การ์ดเดียว** (`listings_public` + `display_listing_id`)
- รหัสกลาง `RXT-YYYY-######` (RealXtate) รวมข้อมูลหลังบ้าน — รองรับ `PPTR-` เก่าในระบบ
- **เจ้าของโพสหลังเอเจ้นท์** → ลำดับติดต่อ 1, remark เจ้าของตรง, แจ้งแอดมิน (`owner_claim_after_agent`)
- เอเจ้นท์ที่โพสก่อน → remark `เอเจ้นท์ — โพสต์ก่อนเจ้าของ`
- ปิด/ไม่ว่างจากประกาศหนึ่ง → ซิงค์ซ่อนประกาศอื่นในทรัพย์เดียวกัน
