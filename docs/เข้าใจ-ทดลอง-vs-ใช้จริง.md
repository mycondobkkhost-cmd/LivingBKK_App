# เข้าใจง่าย: ทดลองบนมือถือ vs ผู้ใช้จริง

## ภาพรวม (สำคัญ)

แอป LivingBKK มี **สองโหมด** — หลายคนสับสนเพราะหน้าจอเหมือนกัน แต่ข้างหลังต่างกัน

| | โหมดทดลอง (Demo) | โหมดใช้จริง (เชื่อม Supabase) |
|--|------------------|-------------------------------|
| **เหมือน** | ดูบ้านบนแผนที่ได้ | ดูบ้านบนแผนที่ได้ |
| **ข้อมูลบ้าน** | ตัวอย่างในแอป (~72 รายการ) | จากคลังข้อมูลบนอินเทอร์เน็ต |
| **ล็อกอิน** | **เลือกบทบาทได้เลย** (ไม่ต้องรหัส) — ดู `docs/โหมดทดลอง.md` | **ต้องมีบัญชี** + รหัสผ่าน |
| **ส่งคำขอสนใจ, ลงประกาศ** | ไม่บันทึกจริง | บันทึกจริง |
| **แอดมิน** | กดเล่นบทบาทในแอปได้ | ต้องเป็นบัญชี admin จริง |

**สรุป:** ถ้ายังล็อกอินไม่ได้ แปลว่ายังไม่เข้าโหมดใช้จริง — แต่ยัง **ดูหน้าจอทดลอง** ได้

---

## ดูว่าตอนนี้อยู่โหมดไหน

แท็บ **ฉัน** ในแอป:

- ขึ้น **「โหมด Demo」** → ยังไม่ต่อคลังข้อมูล (หรือ key ไม่ครบ)
- ขึ้น **「ตั้งค่าแล้ว — กดเลือกบทบาทเพื่อทดลอง」** → ต่อ Supabase แล้ว แต่ยังไม่ได้เลือกบทบาททดลอง
- ขึ้น **「โหมดทดลอง · …」** → เลือกบทบาทแล้ว (UI ครบ บันทึกจริงยังไม่ทำงาน)
- ขึ้น **「ล็อกอินจริง · เชื่อม Supabase」** → ใช้บัญชีจริงแล้ว

---

## ทำไมล็อกอินไม่ได้ (สาเหตุบ่อย)

### 1) ยังไม่มีบัญชีในระบบ

บัญชี `demo-admin@livingbkk.local` **ไม่มีให้เอง** จนกว่าจะรัน **ข้อมูลตัวอย่าง** ใน Supabase  
หรือให้คุณ **สมัครใหม่** ในแอป (ง่ายกว่า)

### 2) ระบบบังคับยืนยันอีเมล

ใน Supabase ต้อง **ปิด Confirm email** ช่วงทดสอบ  
(ดูขั้นตอนด้านล่าง)

### 3) รหัสผิด / พิมพ์อีเมลผิด

ลองสมัครอีเมลใหม่ที่จำง่าย เช่น อีเมลจริงของคุณ + รหัส `Test12345!`

### 4) คลังข้อมูลยังไม่ตั้ง (ตารางใน Supabase)

ต้องรัน `db push` หรือให้คนช่วยตั้ง schema ครั้งหนึ่ง — ไม่งั้นสมัครแล้ว error

---

## วิธีให้ล็อกอินได้ (แนะนำ — สมัครเอง)

### ขั้น A — บนเว็บ Supabase (ทำครั้งเดียว)

1. เข้า https://supabase.com/dashboard → โปรเจกต์ของคุณ  
2. **Authentication** → **Providers** → **Email** → เปิดใช้  
3. **ปิด** ตัวเลือก **Confirm email** (ยืนยันอีเมล) → Save  
4. (ถ้ายังไม่เคยทำ) ให้คนช่วยรัน `db push` บน Mac — สร้างตารางในระบบ

### ขั้น B — บนมือถือ (แอป)

1. แท็บ **ฉัน** → **เข้าสู่ระบบ / สมัคร**  
2. กด **สมัครสมาชิก** (ไม่ใช่เข้าสู่ระบบ)  
3. ใส่อีเมลจริงของคุณ + รหัสผ่าน (อย่างน้อย 6 ตัว)  
4. เลือกบทบาท เช่น **Owner** หรือ **Seeker**  
5. กดสมัคร → ถ้าสำเร็จจะกลับหน้าแรก  
6. แท็บ **ฉัน** ควรขึ้นว่า **ล็อกอินแล้ว**

### ขั้น C — อยากเป็นแอดมิน

หลังสมัครแล้ว ต้องให้คนช่วยรันคำสั่งใน Supabase (SQL):

```sql
UPDATE profiles SET role = 'admin'
WHERE id = (SELECT id FROM auth.users WHERE email = 'อีเมลที่คุณสมัคร');
```

หรือใช้บัญชี demo หลังใส่ seed — [บัญชี-admin.md](บัญชี-admin.md)

---

## หลังล็อกอินแล้ว อะไรใช้ได้จริง

| ฟังก์ชัน | ใช้ได้จริงเมื่อล็อกอิน |
|---------|------------------------|
| ดูแผนที่ / ค้นหา | ได้ (มีข้อมูลจริงถ้า seed แล้ว) |
| กดสนใจทรัพย์ / Lead | บันทึกเข้าระบบ |
| ลงประกาศ | บันทึก (ต้องบทบาท Owner/Agent) |
| แอดมิน | ต้อง role admin |
| แจ้งเตือน | ในแอป (Realtime) |

---

## ผู้ใช้จริงในอนาคตจะใช้ยังไง

1. เปิดลิงก์แอป (เว็บหรือ App Store ภายหลัง)  
2. **สมัคร** ด้วยอีเมลตัวเอง  
3. ใช้งานตามบทบาท (หาบ้าน / ลงประกาศ / เอเจ้นท์)  
4. ทีมคุณดู Lead ใน **ศูนย์ Admin**

ตอนนี้คุณทดบนมือถือ = **เหมือนผู้ใช้จริง** ถ้าล็อกอินสำเร็จ + Supabase ตั้งครบแล้ว
