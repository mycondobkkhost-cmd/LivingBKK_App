# สมุดโครงการจาก Property Hub เว็บเดียว

**สำหรับคุณ (ภาษาคนบ้านๆ):** เราไม่ให้คุณรวบ Excel แล้วอัปไฟล์  
ระบบเข้าเว็บ **Property Hub** ให้ แล้วคัดลอกเฉพาะสิ่งที่แอปต้องใช้มาเก็บในคลาวด์

> **นโยบายพื้นที่:** เฉพาะ **กรุงเทพฯ + ปริมณฑล** (นนทบุรี ปทุมธานี สมุทรปราการ สมุทรสาคร นครปฐม) — ไม่ดึงเชียงใหม่ ภูเก็ต ฯลฯ

---

## ข้อมูลที่ดึงมา (ต่อ 1 โครงการ)

| สิ่งที่ได้ | ใช้ทำอะไร |
|-----------|-----------|
| ชื่อไทย / อังกฤษ | ตอนลงประกาศ พิมพ์แล้วเจอ |
| เขต | แสดงในรายการ / ค้นหา |
| จุดบนแผนที่ (lat/lng) | ปักหมุด |
| BTS/MRT ใกล้เคียง | แสดงในรายละเอียด |
| ประเภท (คอนโด/บ้าน ฯลฯ) | กรองประเภททรัพย์ |
| ปีสร้างเสร็จ | ข้อมูลโครงการ |
| สิ่งอำนวยความสะดวก | สระ, ฟิตเนส, ที่จอด ฯลฯ |
| รูปปก | การ์ดโครงการ |
| ชื่อเล่น (alias) | ค้นหา “ทรู” แล้วเจอ The Trendy |
| ลิงก์ต้นทาง | ย้อนกลับเว็บได้ |

**ไม่ดึง:** ราคาเช่า, เบอร์โทร, ประกาศห้อง — อันนั้นเป็นคนละเรื่อง (แท็บ「นำเข้า」LI)

## ในแอป (แอดมิน)

1. ล็อกอินแอดมิน → แท็บ **โครงการ**
2. กดปุ่มใหญ่ **「ดึงทั้งหมดจาก Property Hub」**
3. รอจนข้อความบอกเสร็จ (อาจ **10–30 นาที** ถ้าโครงการเยอะ)
4. ลองค้นหาชื่อ เช่น `ไลฟ์ อโศก` ในรายการ

### ถ้ามีโครงการใหม่ทีหลัง

- วางลิงก์ `propertyhub.in.th/projects/...` แล้วกด **ดึงและบันทึก**  
  หรือกด **ดึงทั้งหมด** อีกครั้ง (จะอัปเดตของเดิมด้วย)

## วิธีค้นหาโครงการให้ครบ (v2 — ใช้งานจริงได้)

สคริปต์เก่า (`discover-propertyhub-slugs.py`) อ่าน HTML แบบ regex และหยุดที่ 60 หน้า — **ไม่ครบ**  
เช่น Hyde Heritage มีบนเว็บ แต่สคริปต์เก่าไม่เจอ

### วิธีใหม่: `discover-metro-projects-v2.py`

อ่าน **JSON จริง** จากหน้าประกาศ (`__NEXT_DATA__`) แล้วดึง `project.slug` จากทุกประกาศ

| ชั้น | สิ่งที่ไล่ | ตัวอย่าง |
|-----|-----------|---------|
| **A** | เขตกรุงเทพ 47 เขต × 6 ประเภท (คอนโด/บ้าน/ทาวน์ เช่า+ขาย) | `/en/condo-for-rent/watthana-bangkok` (632 หน้า) |
| **B** | โซน BTS + MRT | `/en/condo-for-rent/bts-asok`, `/en/condo-for-rent/mrt-sukhumvit` |
| **C** | 6 จังหวัดปริมณฑลเต็มจังหวัด | นนทบุรี ปทุมธานี สมุทรปราการ ฯลฯ |
| **D** | โครงการ seed + หน้า `/projects/{slug}` + `/en/condo-for-rent/project-{slug}` | Hyde Heritage, Ashton Asoke ฯลฯ |
| **E** | **ค้นหาตามตัวอักษร** — พิมพ์ `?text=` บนหน้าประกาศทุกจังหวัดปริมณฑล | `a`–`z`, `hyde`, `ไลฟ์`, `ทรู`, ทองหล่อ ฯลฯ |

ชั้น **E** ทำหน้าที่เหมือนพิมพ์ในช่องค้นหาของเว็บ — จับโครงการที่ชั้น A อาจพลาด (เช่น Hyde Heritage)

**คาดว่าได้:** 2,000–5,000+ ชื่อโครงการ (กทม.+ปริมณฑล) หลังไล่ครบทุกชั้น  
**เวลา:** ประมาณ **3–10 ชั่วโมง** (ชั้น E เพิ่ม ~1–2 ชม.)

### รันเต็ม (แนะนำ)

```bash
source scripts/dev-path.sh
./scripts/deploy-projects-cloud.sh
./scripts/rediscover-metro-projects.sh
```

### รันทีละขั้น

```bash
# 1) ค้นหาชื่อ (บันทึก /tmp/ph-metro-slugs.json — resume ได้)
python3 scripts/discover-metro-projects-v2.py

# 2) ปิดโครงการนอกพื้นที่
./scripts/purge-non-metro-projects.sh

# 3) ดึงรายละเอียดขึ้น Cloud
SKIP_DISCOVER=1 ./scripts/sync-propertyhub-cloud.sh
```

### ทดสอบสั้นๆ (smoke test)

```bash
RESUME=0 SMOKE_TEST=1 MAX_PAGES_PER_SEED=2 MAX_PAGES_PER_QUERY=2 python3 scripts/discover-metro-projects-v2.py
```

### ตัวแปรที่ปรับได้

| ตัวแปร | ความหมาย |
|--------|----------|
| `MAX_PAGES_PER_SEED=100` | จำกัดหน้าต่อ seed ชั้น A-C (0 = ไม่จำกัด) |
| `MAX_PAGES_PER_QUERY=8` | จำกัดหน้าต่อคำค้น ชั้น E |
| `LAYERS=A,B,C,E` | เลือกชั้น (ค่าเริ่มต้น A,B,C,D,E) |
| `RESUME=0` | เริ่มใหม่ทั้งหมด |
| `SMOKE_TEST=1` | ทดสอบสั้นๆ (seeds + 6 คำค้น) |

> **หมายเหตุ:** ชื่อโครงการ ≠ ประกาศห้อง — หลังได้สมุดโครงการแล้ว ต้องนำเข้า listings แยกต่างหาก

## สมุดแสดงผลดรอปดาวน์ (ค้นตามตัวอักษร)

เก็บ「สิ่งที่เว็บแสดงตอนพิมพ์」ไว้ในแอป — เช่น พิมพ์ **ห** → **ห้วยขวาง**, โครงการ, โซน ฯลฯ  
**เก็บครบก่อน** แล้วค่อยกรองทีหลัง (ตอนนี้ยังไม่ตัดของนอกพื้นที่ใน index นี้)

| ส่วน | ไฟล์ |
|------|------|
| สคริปต์เก็บข้อมูล | `scripts/discover-search-display-index.py` |
| ข้อมูลในแอป | `mobile/assets/data/search_display_index.json` |
| โหลดในแอป | `SearchDisplayCatalog` → ผสมใน `SearchService.suggest()` |

### รัน

```bash
# bootstrap เขต/BTS/MRT + crawl จากเว็บ ?text=
python3 scripts/discover-search-display-index.py

# ทดสอบสั้น (6 คำค้น)
SMOKE_TEST=1 python3 scripts/discover-search-display-index.py

# bootstrap อย่างเดียว (ไม่ยิงเว็บ)
CRAWL_WEB=0 python3 scripts/discover-search-display-index.py
```

รวมใน `./scripts/rediscover-metro-projects.sh` เป็นขั้น 3 อัตโนมัติ

### ในแอป

- ดรอปดาวน์แสดงผลตั้งแต่ **1 ตัวอักษร** (หน้าค้นหา + SmartSearchBar)
- กดทำเล → เปิดรายการในพื้นที่ | กดโครงการ → หน้าห้องในโครงการ

## ทำไมไม่ดึงครั้งเดียวทีเดียวหมดพันรายการ?

เว็บและเซิร์ฟเวอร์จะ **ค้างหรือตัด** ถ้ายิงเร็วเกินไป  
เลยดึงทีละชุด ~50 รายการ แล้ววนจนครบ — แอปทำให้อัตโนมัติหลังกดปุ่มเดียว
