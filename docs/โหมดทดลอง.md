# โหมดทดลอง (TRIAL_MODE)

ช่วงทดสอบแอป LivingBKK สามารถ **เลือกบทบาทแล้วเข้าได้ทันที** โดยไม่ต้องมีรหัสผ่านหรือ seed บัญชีบน Supabase

## วิธีใช้

1. เปิดแอป (Chrome หรือมือถือผ่าน `./scripts/run-for-phone.sh`)
2. แท็บ **ฉัน** → **เข้าสู่ระบบ / ทดลอง** → **เข้าทดลอง (ไม่ต้องรหัส)**  
   หรือล็อกอินจริงด้วยอีเมล/รหัส (สมัครครั้งเดียว ไม่เลือกบทบาท)
3. แท็บ **หน้าแรก** → แถบ **「คุณเป็นใคร」** สลับ:
   - ลูกค้า · หาซื้อ/เช่าเอง
   - เอเจ้นท์ · หาทรัพย์รับโค (เห็นเฉพาะทรัพย์รับโค)
   - เจ้าของ · ลงประกาศ (มุมมองดูบ้านเหมือนลูกค้า + ปุ่มลงประกาศ)

**แอดมินระบบ** แยกต่างหาก — ไม่สลับจากแถบนี้ (ต้องเป็นบัญชี admin ใน Supabase)

## สิ่งที่ทำได้ / ทำไม่ได้

| ทำได้ | ยังไม่ทำ (จนกว่าจะปิดโหมดทดลอง) |
|--------|----------------------------------|
| สลับแท็บ ดูแผนที่ บอร์ด งาน Admin UI | บันทึก Lead / ลงประกาศ / แก้ข้อมูลบน Supabase จริง |
| ทดสอบฟอร์มนัดดู แชทจำลอง กล่อง Lead ตัวอย่าง | Push notification ผูกกับบัญชีจริง |

เมื่อบันทึกในช่วงทดลอง แอปจะแสดงข้อความ **「โหมดทดลอง — ยังไม่บันทึกลงระบบจริง」**

## โหมดทดลองแยก (หลังบ้าน + สลับตัวละคร)

เมื่อล็อกอิน `demo-admin@livingbkk.local` และเปิด `ADMIN_DEMO_CASES=true` แอปจะเข้า **โหมดทดลองแยก**:

- นัดดู / แชท / เคส / KPI บนแดชบอร์ด — จากข้อมูลจำลองเท่านั้น
- **ไม่ดึง inbox หรือตัวเลข KPI จาก Supabase** มาผสม (แก้ปัญหา badge ไม่ตรง / แชทหายหลังยืนยันนัด)
- แถบสีฟ้า: 「โหมดทดลองแยก — ข้อมูลจำลองทั้งหมด ไม่ผสม DB」

ก่อน production จริง: ตั้ง `ADMIN_DEMO_CASES=false` (และ `TRIAL_MODE=false`) แล้วทดสอบด้วยข้อมูล DB อย่างเดียว

**เคลียร์เคสหลังทดสอบ:** หน้า **ปฏิทินนัดชม** → ปุ่ม **「เคลียร์เคสทดลอง」** — รีเซ็ตนัด/เอเจ้นที่มอบหมาย/แชทจำลองกลับค่าเริ่มต้น (มีเคสรอมอบหมายให้ลองใหม่)

## ตั้งค่า

ไฟล์ `mobile/assets/env`:

```env
TRIAL_MODE=true
ADMIN_DEMO_CASES=true
```

- `TRIAL_MODE=true` (ค่าเริ่มต้น) = เข้าทดลองไม่ต้องรหัสได้
- `TRIAL_MODE=false` = ต้องล็อกอินจริง (พร้อมเปิดใช้งานจริง)
- `ADMIN_DEMO_CASES=true` = เคสตัวอย่างหลังบ้าน + โหมดทดลองแยกเมื่อใช้บัญชี demo-admin
- `ADMIN_DEMO_CASES=false` = หลังบ้านอ่านจาก DB อย่างเดียว

หลังแก้ไฟล์ รันแอปใหม่หรือกด **R** (hot restart)

## เมื่อพร้อมเปิดใช้งานจริง

1. ตั้ง `TRIAL_MODE=false` ใน `mobile/assets/env`
2. รัน seed บน Supabase (`./scripts/seed-cloud.sh`) และบัญชี demo ตาม `docs/บัญชี-admin.md`
3. ปิด Confirm email ใน Supabase ถ้าต้องการทดสอบล็อกอินเร็ว — ดู `docs/เข้าใจ-ทดลอง-vs-ใช้จริง.md`
4. หน้าเข้าสู่ระบบจะเหลือเฉพาะ **ล็อกอินจริง**

## ล็อกอินจริงระหว่างทดลอง

ในหน้าเข้าสู่ระบบ ยังมีส่วน **「ล็อกอินจริง (เมื่อพร้อมเปิดใช้งาน)」** แบบพับได้ — ใช้ทดสอบ Supabase คู่กับบัญชี seed ได้โดยไม่ต้องปิด `TRIAL_MODE` ก่อน
